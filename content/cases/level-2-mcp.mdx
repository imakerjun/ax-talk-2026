# MCP stdio로 AI 자산 헤쳐모여

> 참고: [흩어져 있는 AI 자산, 'MCP stdio'로 헤쳐모여!](https://techblog.woowahan.com/25986/) (조현석, 2026.03)

---

## 문제

- 프롬프트, 스킬, 규칙이 개인 메모장에 흩어져 있음
- 잘 쓰는 사람의 노하우가 공유는 되지만, 다른 사람이 그대로 쓸 수 없음
- IDE, 도구가 바뀌면 자산도 사라짐

---

## 기존 방식 vs MCP stdio

| | Agent Sync (기존) | MCP stdio |
|---|---|---|
| 자산 위치 | 각 IDE 설정 파일에 분산 | 중앙 패키지 하나에 집중 |
| 공유 방식 | 문서로 공유 → 각자 복사 | `npm install` → 자동 적용 |
| IDE 종속 | Cursor rules, Copilot instructions 등 각각 관리 | IDE 무관, 어디서든 동일 |
| 버전 관리 | 없음 (최신이 어디인지 모름) | npm 버전으로 이력 추적 |
| 업데이트 | 공지 → 각자 수동 반영 | `npm update` 한 줄 |

---

## 해결: MCP stdio 기반 중앙화 + npm 배포

패키지 구조:

```
central-mcp-agent/
├── src/
│   ├── prompts/          # 팀 공용 프롬프트
│   ├── skills/           # 재사용 가능한 스킬
│   ├── rules/            # 코드 컨벤션, 리뷰 규칙
│   └── index.ts          # MCP stdio 서버 엔트리
├── package.json
└── README.md
```

- 팀의 AI 자산(프롬프트, 스킬, 규칙)을 하나의 패키지로 관리
- `npm install` → Cursor, Claude Code 등 어떤 IDE에서든 자동으로 최신 자산 적용
- 버전 관리로 자산의 변경 이력 추적
