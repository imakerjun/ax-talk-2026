# 배달의민족 디자인팀 — 스컬피 원형으로 AI 디자인 자산 만들기

## 문제

- 배달의민족만의 독보적인 비주얼 언어를 일관되게 유지하면서 디자인 자산을 대량 생산해야 함
- AI로 이미지를 생성하면 "배민스러움"이 사라짐 — 일반적인 AI 결과물이 나옴
- 디자이너가 하나하나 직접 만들면 시간이 너무 오래 걸림

## AI로 디자인을 만들 때 중요한 사고 방식

> 일관된 비주얼 언어로 '디자인 자산'을 만들 수 있어야 한다.

핵심 질문: **딱 하나의 에센스를 넣는다면?**

## 해결: "에센스를 집어넣기" — 스컬피 원형 + AI 생성

1. 디자이너가 직접 **스컬피(찰흙)로 원형을 조각** — 배민의 비주얼 에센스를 물리적으로 정의
2. 이 원형 사진을 Adobe Firefly에 **스타일 참조**로 입력
3. "오토바이 그려줘" 같은 간단한 프롬프트 → 배민 스타일의 3D 오브젝트가 생성됨
4. 포토샵에서 색상·디테일 보정 → 캠페인 포스터까지 완성

## 결과

- 스컬피 원형 하나로 바나나, 배달봉투, 우유팩, 오토바이 등 **수십 개의 디자인 자산** 생성
- "배민스러움"이 일관되게 유지됨 — AI가 만들어도 브랜드 정체성이 살아있음
- 디자인 자산 생산 속도가 비약적으로 향상

## 포인트

자산화의 핵심은 **에센스를 정의하는 것**이다.

- 개발팀은 코드 컨벤션·프롬프트를 패키지로
- 디자인팀은 비주얼 원형을 스타일 참조로

직군이 달라도 구조는 같다. "AI에게 무엇을 시킬까"가 아니라 **"AI에게 어떤 기준을 줄까"**가 자산화의 본질.

**에센스가 기준이 되면 — 그게 축적이다.**
