# 다른 기업들의 축적

> 우아한형제들만의 이야기가 아닙니다. 같은 여정을 걷고 있는 기업들이 있습니다.

---

## 우아한형제들 — 물어보새

사내 해커톤(우아톤 2023) 1등 → 정식 서비스화된 **AI 데이터 분석가 슬랙봇**

| 항목 | 내용 |
|------|------|
| 핵심 기능 | 자연어로 데이터 조회 (Text-to-SQL), RAG 기반 사내 지식 검색 |
| 기술 | LLM + RAG + LangChain + 벡터 스토어 |
| 인터페이스 | **슬랙** — 별도 UI 없이 대화창에서 바로 사용 |

### 성과

- 전사 구성원 **33%**가 업무에 활용
- 사용자 만족도 **95%** 긍정 평가
- 조직별 문의 해결 속도 평균 **21% 향상**
- AWS re:Invent 2025에서 사례 발표

> 참고: [AI 데이터 분석가 '물어보새' 등장 — 1부. RAG와 Text-to-SQL](https://techblog.woowahan.com/18144/) / [2부. Data Discovery](https://techblog.woowahan.com/18362/)

## 당근 — 카비 (KABI)

GenAI 플랫폼 위에서 동작하는 **Claude Code 기반 사내 AI 에이전트 슬랙봇**

| 항목 | 내용 |
|------|------|
| 핵심 기능 | 질문자의 맥락(소속팀, 접근 가능 저장소) 파악 → 맞춤 응답 |
| 기술 | Claude Code + MCP + GenAI 플랫폼(LLM Router + Prompt Studio) |
| 인터페이스 | **슬랙** |

### 실제 활용

- 디자이너가 코드를 조회하고 PR을 올림
- 제품팀 리드가 복잡한 버그 원인을 카비로 빠르게 파악
- **비개발자도 슬랙에서 어드민 기능을 직접 개발**할 수 있는 수준까지

> 참고: [클로드 코드 쓰다가 결국 '이 도구'로 갈아탄 이유 — 당근 하조은님 (YouTube)](https://www.youtube.com/watch?v=88k-kIl45Tk)

## 오늘의집 — 오집사

Amazon Bedrock 기반의 **사내 AI 인턴 슬랙봇**

| 항목 | 내용 |
|------|------|
| 핵심 기능 | 아이디어 논의, 대화 요약, 사내 부서별 문서 검색 |
| 기술 | Amazon Bedrock Agent + Knowledge Bases + OpenSearch |
| 인터페이스 | **슬랙** 커맨드로 호출 |

### 특징

- 각 팀이 데이터만 준비하면 에이전트를 빠르게 추가할 수 있는 **멀티에이전트 구조**
- 페르소나: "오늘의집 인턴" — 탭 전환 없이 슬랙 안에서 바로 사용

> 참고: [오늘의집 AI 인턴 '오집사' 개발 여정](https://www.bucketplace.com/post/2025-01-17-오늘의집-ai-인턴-오집사-개발-여정/) / [AWS 기술 블로그 — 오집사와 Amazon Bedrock](https://aws.amazon.com/ko/blogs/tech/journey-ozipsa-with-amazon-bedrock/)

## 공통 패턴: "축적"의 구조가 보인다

| 패턴 | 사례 |
|------|------|
| **인터페이스는 슬랙** | 물어보새, 카비, 오집사 — 모두 별도 UI가 아닌 이미 쓰는 도구에 녹임 |
| **RAG로 사내 지식 연결** | LLM 단독이 아니라 내부 데이터와 결합해야 실무에서 쓸 수 있음 |
| **해커톤에서 시작** | 물어보새(우아톤 1등), 당근(Gen AI 해커톤) — 실험에서 시작해서 쓸 만하면 투자 |
| **비개발자까지 확장** | 카비(디자이너가 PR), 물어보새(SQL 모르는 구성원), 오집사(팀별 확장) |

> 어떤 기업이든 핵심은 같습니다 — **"개인의 AI 사용 → 팀/조직의 AI 활용"으로 전환하는 구조를 만드는 것.**
